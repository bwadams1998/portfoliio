Amazon E-commerce Test Cases

Test Case 1: User Login - Valid Credentials

Test ID: AMZ-TC-001
Priority: High
Module: Authentication

Preconditions:

  • User has a valid Amazon account
  • User is on Amazon homepage

Test Steps:

  1. Navigate to www.amazon.com
  2. Click on "Sign in" button in top right corner
  3. Enter valid email/phone number
  4. Click "Continue"
  5. Enter valid password
  6. Click "Sign in"

Expected Result:

  • User successfully logs in
  • Redirected to Amazon homepage or previous page
  • User name appears in top right corner
  • "Hello, [User Name]" greeting displayed

Test Data:

  • Email: testuser@example.com
  • Password: ValidPass123!

Test Case 2: Product Search Functionality

Test ID: AMZ-TC-002
Priority: High
Module: Search

Preconditions:

  • User is on Amazon homepage

Test Steps:

  1. Locate search bar at top of page
  2. Enter product name "wireless headphones"
  3. Click search icon or press Enter

Expected Result:

  • Search results page loads within 2-3 seconds
  • Results display relevant products matching search query
  • Filters appear on left sidebar (Brand, Price, Reviews, etc.)
  • Number of results shown (e.g., "1-48 of over 5,000 results")
  • Products display with image, title, price, and rating

Test Data:

  • Search Query: "wireless headphones"

Test Case 3: Add Item to Cart

Test ID: AMZ-TC-003
Priority: High
Module: Shopping Cart

Preconditions:

  • User is logged in
  • User is viewing a product detail page

Test Steps:

  1. Navigate to any product page
  2. Verify "Add to Cart" button is visible
  3. Select quantity (if applicable)
  4. Select product options (size, color, etc.) if required
  5. Click "Add to Cart" button
  6. Observe confirmation message

Expected Result:

  • "Added to Cart" confirmation appears
  • Cart icon in header updates with item count
  • Option to "Proceed to checkout" or "Continue shopping" displayed
  • Product successfully appears in cart when viewing cart page

Test Data:

  • Product: Any eligible product
  • Quantity: 1

Test Case 4: Apply Promo Code at Checkout

Test ID: AMZ-TC-004
Priority: Medium
Module: Checkout

Preconditions:

  • User is logged in
  • At least one item in cart
  • Valid promo code available

Test Steps:

  1. Click on cart icon
  2. Click "Proceed to Checkout"
  3. Verify shipping address
  4. Continue to payment page
  5. Locate "Add promo code" or "Gift card/Promo code" field
  6. Enter valid promo code
  7. Click "Apply"

Expected Result:

  • Promo code validates successfully
  • Discount applied to order total
  • Updated price displayed
  • Confirmation message "Your coupon has been applied"
  • Order summary reflects discount

Test Data:

  • Promo Code: TEST10 (example)

Test Case 5: Filter Search Results by Price

Test ID: AMZ-TC-005
Priority: Medium
Module: Search/Filters

Preconditions:

  • User has performed a product search
  • On search results page

Test Steps:

  1. Perform search for "laptop"
  2. Locate price filter on left sidebar
  3. Select price range "$500 to $1000"
  4. Observe results update

Expected Result:

  • Page refreshes or updates dynamically
  • All displayed products fall within selected price range
  • Filter selection highlighted/checked
  • URL updates to reflect filter parameters
  • Product count updates

Test Data:

  • Search Term: "laptop"
  • Price Range: $500-$1000
