Test Case 6: Gmail Login - Valid Credentials

Test ID: GMAIL-TC-001
Priority: High
Module: Authentication

Preconditions:

  • User has valid Gmail account
  • User is on Gmail login page

Test Steps:

  1. Navigate to gmail.com
  2. Enter valid email address
  3. Click "Next"
  4. Enter valid password
  5. Click "Next"

Expected Result:

  • User successfully logs in
  • Gmail inbox loads
  • Email count displayed
  • Navigation menu visible (Inbox, Sent, Drafts, etc.)
  • Profile picture/initial appears in top right

Test Data:

  • Email: testuser@gmail.com
  • Password: SecurePass123!

Test Case 7: Compose and Send Email

Test ID: GMAIL-TC-002
Priority: High
Module: Email Composition

Preconditions:

  • User is logged into Gmail
  • In inbox view

Test Steps:

  1. Click "Compose" button
  2. Enter recipient email in "To" field
  3. Enter subject line
  4. Type message in body
  5. Click "Send" button

Expected Result:

  • Compose window opens
  • All fields are editable
  • Email sends successfully
  • "Message sent" confirmation appears
  • Email appears in "Sent" folder
  • Compose window closes automatically

Test Data:

  • To: recipient@example.com
  • Subject: Test Email
  • Body: This is a test message.

Test Case 8: Search Emails by Keyword

Test ID: GMAIL-TC-003
Priority: Medium
Module: Search

Preconditions:

  • User is logged in
  • User has emails in inbox

Test Steps:

  1. Locate search bar at top of page
  2. Enter search keyword (e.g., "invoice")
  3. Press Enter or click search icon
  4. Review search results

Expected Result:

  • Search executes within 1-2 seconds
  • Results show emails containing keyword
  • Search term highlighted in results
  • Results display sender, subject, date, and snippet
  • Can filter results by date, sender, attachment, etc.

Test Data:

  • Search Term: "invoice"

Test Case 9: Attach File to Email

Test ID: GMAIL-TC-004
Priority: Medium
Module: Email Composition

Preconditions:

  • User is logged in
  • Compose window is open

Test Steps:

  1. Click "Compose"
  2. Click attachment icon (paperclip)
  3. Select "Attach files"
  4. Choose file from computer (under 25MB)
  5. Wait for upload to complete
  6. Add recipient and send email

Expected Result:

  • File browser opens
  • Selected file uploads successfully
  • Upload progress bar shows completion
  • File name appears in compose window with file size
  • Email sends with attachment
  • File size under 25MB limit accepted

Test Data:

  • File: TestDocument.pdf (2MB)

Test Case 10: Mark Email as Spam

Test ID: GMAIL-TC-005
Priority: Medium
Module: Email Management

Preconditions:

  • User is logged in
  • At least one email in inbox

Test Steps:

  1. Open or select an email
  2. Click "Report spam" icon or button
  3. Observe email behavior

Expected Result:

  • Email immediately moves to Spam folder
  • Confirmation message appears
  • Email removed from inbox
  • Option to "Undo" appears briefly
  • Future emails from sender may be auto-filtered

Test Data:

  • Email: Any email in inbox
