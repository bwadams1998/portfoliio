Test Case 16: Discord Login - Valid Credentials

Test ID: DISC-TC-001
Priority: High
Module: Authentication

Preconditions:

  • User has valid Discord account
  • User is on Discord login page

Test Steps:

  1. Navigate to discord.com/login
  2. Enter valid email or phone number
  3. Enter valid password
  4. Click "Log In" button

Expected Result:

  • User successfully logs in
  • Discord home screen/server list loads
  • User's avatar and username displayed
  • Server list appears on left sidebar
  • Friends list or last viewed channel opens
  • Login session persists

Test Data:

  • Email: testuser@example.com
  • Password: DiscordPass123!

Test Case 17: Send Direct Message

Test ID: DISC-TC-002
Priority: High
Module: Messaging

Preconditions:

  • User is logged in
  • User has at least one friend or contact

Test Steps:

  1. Click on "Friends" icon in left sidebar
  2. Select a friend from list
  3. Type message in text input field at bottom
  4. Press Enter or click Send icon

Expected Result:

  • Message appears in chat window immediately
  • Message shows user avatar and username
  • Timestamp displayed
  • Message delivered to recipient
  • "Typing..." indicator may appear for recipient
  • Message persists in chat history

Test Data:

  • Message: "Hello, this is a test message"

Test Case 18: Join Server via Invite Link

Test ID: DISC-TC-003
Priority: Medium
Module: Server Management

Preconditions:

  • User is logged in
  • User has valid invite link

Test Steps:

  1. Click "+" button in server list (left sidebar)
  2. Select "Join a Server"
  3. Paste invite link in text field
  4. Click "Join Server" button

Expected Result:

  • Server validation occurs
  • User joins server successfully
  • Server appears in server list
  • User redirected to server's default channel
  • Welcome message may appear (if configured)
  • Server roles assigned (if applicable)

Test Data:

  • Invite Link: discord.gg/example123

Test Case 19: Create Voice Channel

Test ID: DISC-TC-004
Priority: Medium
Module: Voice/Channel Management

Preconditions:

  • User is logged in
  • User has admin/manage channel permissions on a server
  • User is viewing server

Test Steps:

  1. Navigate to server where user has permissions
  2. Right-click on server name or channel category
  3. Select "Create Channel"
  4. Choose "Voice Channel" type
  5. Enter channel name
  6. Click "Create Channel"

Expected Result:

  • Create channel dialog opens
  • Voice channel option selectable
  • Channel name field accepts input
  • New voice channel appears in server channel list
  • Channel is accessible to users based on permissions
  • Voice channel icon (speaker) displayed

Test Data:

  • Channel Name: "Test Voice Chat"

Test Case 20: React to Message with Emoji

Test ID: DISC-TC-005
Priority: Low
Module: User Interaction

Preconditions:

  • User is logged in
  • User is viewing a text channel with messages

Test Steps:

  1. Hover over any message in a channel
  2. Click on "Add Reaction" icon (emoji face with plus sign)
  3. Select an emoji from picker
  4. Observe reaction appear

Expected Result:

  • Emoji picker opens
  • Searchable emoji list displayed
  • Selected emoji appears below message
  • Reaction count shows "1"
  • Clicking same emoji again adds to count
  • User can remove reaction by clicking their reaction
  • Reactions persist and visible to all users
